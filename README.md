# Bitrix Importer

GUI-приложение для импорта Excel-файлов в Bitrix24 через входящий webhook.

Приложение умеет:
- подключаться к Bitrix24 по webhook
- показывать статусы лидов
- показывать воронки сделок и стадии по ним
- загружать Excel в `лиды` или `сделки`
- создавать контакты и компании в CRM или писать все контактные данные в комментарий
- сохранять отчет об импорте в `.xlsx`
- собираться в `.exe` для Windows

## Возможности

- Поле для ввода и сохранения `BITRIX_URL_WEBWORKER`
- Просмотр справочников CRM прямо из GUI
- Выбор:
  - `lead` или `deal`
  - нужной воронки и стадии сделки
  - статуса лида
  - режима загрузки контактов
- Выбор Excel-файла через диалог
- Предпросмотр колонок и первых строк файла
- Журнал работы в окне приложения

## Структура проекта

- `main.py` — GUI-приложение
- `bitrix_importer.py` — логика импорта и работа с Bitrix API
- `test.py` — консольный просмотр статусов лидов и воронок сделок
- `build_exe.bat` — сборка Windows `.exe`

## Требования

- Python 3.11+  
- Windows
- Доступный входящий webhook Bitrix24

Python-зависимости:
- `pandas`
- `requests`
- `python-dotenv`
- `openpyxl`
- `pyinstaller` — только для сборки `.exe`

## Установка

```bash
pip install pandas requests python-dotenv openpyxl
```

Для сборки `.exe`:

```bash
pip install pyinstaller
```

## Настройка webhook

Создайте файл `.env` в корне проекта:

```env
BITRIX_URL_WEBWORKER=https://your-company.bitrix24.ru/rest/1/your_webhook/
```

Можно не редактировать `.env` вручную: webhook также сохраняется из GUI.

## Запуск

```bash
python main.py
```

После запуска:
1. Вставьте webhook
2. Нажмите `Показать лиды` или `Показать сделки`
3. Выберите, куда загружать данные
4. Выберите Excel-файл
5. При необходимости выберите режим контактов
6. Нажмите `Запустить импорт`

## Режимы контактов

`entities`
- приложение создаёт компанию и контакт в CRM
- для сделок контакт привязывается к сделке

`comments`
- приложение не создаёт отдельные сущности контакта/компании
- телефон, email, сайт, ФИО и другие данные уходят в комментарий

## Формат Excel

Ожидается Excel-файл с колонками:

- `Название компании (полное)`
- `Название компании (сокращенное)`
- `ОГРН`
- `ИНН`
- `Юр. адрес`
- `Регион`
- `Дата регистрации`
- `ФИО руководителя`
- `Должность руководителя`
- `Основной ОКВЭД (код)`
- `Основной ОКВЭД (описание)`
- `Количество сотрудников`
- `Телефоны`
- `Сайты`
- `Email`
- `Выручка за последний предоставленный период`
- `Прибыль за последний предоставленный период`

Минимально обязательные колонки:
- `Название компании (полное)`
- `ИНН`

## Консольная проверка справочников

Если нужно быстро посмотреть статусы лидов и воронки сделок без GUI:

```bash
python test.py
```

## Сборка `.exe`

Простой способ:

```bat
build_exe.bat
```

Или вручную:

```bash
pyinstaller --noconfirm --onefile --windowed --name BitrixImporter main.py
```

Готовый файл будет в папке `dist/`.

## Отчет об импорте

После загрузки приложение сохраняет `.xlsx`-отчет со статусом по каждой строке:
- номер строки
- `OK` или `ERROR`
- созданные ID сущностей
- текст ошибки, если импорт не удался

## Безопасность

- Не публикуйте реальный webhook в GitHub
- Добавьте `.env` в `.gitignore`
- Если webhook уже попал в публичный репозиторий, перевыпустите его в Bitrix24
